# 📊 Finance & Banking Analytics Portfolio

## 📌 Overview
This portfolio showcases three end‑to‑end projects in the **finance and banking domain**, integrating **SQL, Python, and Power BI**. Each project demonstrates technical depth, business relevance, visual storytelling.

---

## 🛡️ Project 1: Anti-Money Laundering (AML Monitoring System)

### 🔹 Objective
Detect suspicious transactions by combining **rule-based SQL queries** with **Python machine learning models**, and visualize results in **Power BI dashboards**.

### 🔹 Workflow
- **SQL**: Cleaned 9.5M+ records, engineered features (frequency, rolling sums, high‑risk flags), built percentile queries.  
- **Python**: Applied Random Forest, Logistic Regression, and Isolation Forest; handled class imbalance with SMOTE; generated risk scores.  
- **Power BI**: Dashboards with KPIs, transaction trends, suspicious activity tables, and network analysis visuals.  

### 🔹 Outcomes
- Identified structuring/layering patterns and high‑risk jurisdictions.  
- Reduced false positives with ML anomaly detection.  
- Delivered compliance dashboards aligned with FATF standards.  

---

## 💳 Project 2: Online Payment Fraud Detection

### 🔹 Objective
Detect fraudulent online payment transactions in real time using **machine learning classification and anomaly detection**.

### 🔹 Workflow
- **SQL**: Extracted transaction metadata (IP address, device type, geolocation, transaction amount).  
- **Python**:  
  - Preprocessed categorical variables with one‑hot encoding.  
  - Applied supervised models (Logistic Regression, Random Forest, XGBoost).  
  - Used unsupervised anomaly detection (Isolation Forest, Autoencoders).  
  - Balanced imbalanced fraud labels with SMOTE.  
- **Power BI**: Dashboard showing fraud trends, flagged transactions, and fraud probability distribution.  

### 🔹 Outcomes
- Detected fraudulent transactions with high precision.  
- Reduced false positives by combining supervised and anomaly detection models.  
- Provided real‑time fraud monitoring dashboards for compliance teams.  

---

## 📈 Project 3: Credit Risk Analysis

### 🔹 Objective
Predict loan defaults and assess borrower risk to support **credit scoring and risk management**.

### 🔹 Workflow
- **SQL**: Extracted borrower demographics, loan history, repayment behavior.  
- **Python**:  
  - Engineered features (income‑to‑loan ratio, repayment delays, credit utilization).  
  - Applied Logistic Regression, Random Forest, and Gradient Boosting models.  
  - Evaluated models with precision, recall, ROC‑AUC.  
- **Power BI**: Dashboard showing borrower risk segments, default probability distribution, and repayment trends.  

### 🔹 Outcomes
- Improved accuracy of loan default predictions.  
- Enabled proactive risk assessment for lending teams.  
- Visualized borrower risk profiles for decision‑makers.

## 🚀 Business Impact Across Projects
- **Compliance**: AML monitoring reduced false positives and flagged high‑risk activity.  
- **Fraud Prevention**: Online payment fraud detection improved transaction security.
- **Risk Management**: Credit risk analysis enhanced borrower profiling and default prediction.
